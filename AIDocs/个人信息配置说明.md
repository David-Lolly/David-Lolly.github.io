# 个人信息配置使用说明

## 📝 概述

现在您可以通过修改 `content/basic/` 文件夹中的 Markdown 文件来管理网站的个人信息，无需修改代码文件。

## 📂 配置文件结构

```
content/basic/
├── profile.mdx    # 个人基本信息（首页展示）
└── about.mdx      # 关于页面内容
```

## 🔧 配置文件说明

### 1. profile.mdx - 个人基本信息

此文件控制首页个人信息卡片的内容。

**字段说明：**

```yaml
---
type: profile          # 固定值，不要修改
name: 熊乐乐           # 您的姓名
avatar: /static/images/basic_info/avatar.jpg  # 头像路径
position: 开发工程狮    # 职位/身份
bio: 一只开发工程狮...  # 个人简介（1-2句话）
location: 中国         # 所在地
email: xxx@qq.com     # 邮箱
github: https://github.com/xxx    # GitHub 链接
twitter:              # Twitter 链接（可选）
linkedin:             # LinkedIn 链接（可选）
website: https://www.mcaoyuan.com  # 个人网站（可选）
tech_stack:           # 技术栈列表（可选）
  - Python
  - FastAPI
  - LangChain
  - Docker
  - Vue
  - AI/ML
---
```

**使用示例：**

修改 `content/basic/profile.mdx` 中的字段，保存后网站会自动更新首页的个人信息卡片。

**注意事项：**
- 所有链接字段（email, github, twitter, linkedin, website）如果不需要可以留空或删除
- tech_stack 字段如果不需要展示技术栈，可以删除整个字段
- avatar 路径建议使用 `/static/images/` 下的图片

### 2. about.mdx - 关于页面

此文件控制 `/about` 页面的内容。

**frontmatter 字段：**

```yaml
---
type: about      # 固定值，不要修改
title: 关于我     # 页面标题
slug: about      # URL slug
---
```

**正文内容：**

frontmatter 之后的所有 Markdown 内容都会渲染在关于页面。

**使用示例：**

```markdown
---
type: about
title: 关于我
slug: about
---

## About

Hello！欢迎访问我的博客...

## 技术栈

我主要使用以下技术：

- **后端**: Python, FastAPI
- **前端**: Vue.js, React

## 联系方式

- Email: xxx@qq.com
- GitHub: xxx
```

## 🎯 效果展示

### 首页个人信息卡片会显示：
- ✅ 头像
- ✅ 姓名
- ✅ 职位
- ✅ 个人简介
- ✅ 技术栈（可选）
- ✅ 社交链接（GitHub, Twitter, LinkedIn, Email, Website）

### 关于页面会显示：
- ✅ 页面标题
- ✅ 完整的 Markdown 内容（支持标题、列表、代码块、图片等）

## 🚀 如何更新

1. **修改个人信息**：编辑 `content/basic/profile.mdx`
2. **修改关于内容**：编辑 `content/basic/about.mdx`
3. **保存文件**：保存后 Velite 会自动重新构建
4. **刷新浏览器**：查看更新后的效果

## 💡 高级使用

### 更换头像

1. 将新头像放到 `public/static/images/basic_info/` 文件夹
2. 修改 `profile.mdx` 中的 `avatar` 字段指向新图片

### 添加新的社交链接

目前支持的社交链接图标：
- GitHub (github)
- Twitter (twitter)
- LinkedIn (linkedin)
- Email (email)
- Website (website)

如需添加其他社交平台，需要修改 `components/profile-card.tsx` 组件。

### 关于页面排版

关于页面支持完整的 Markdown 语法：
- 标题 (# ## ###)
- 列表（无序、有序）
- 代码块
- 引用
- 表格
- 图片
- 链接

## 📌 注意事项

1. **不要修改 `type` 字段**：这是用于区分配置类型的标识
2. **链接格式**：确保 URL 以 `http://` 或 `https://` 开头
3. **图片路径**：使用相对路径或绝对路径
4. **保存编码**：确保文件以 UTF-8 编码保存

## 🐛 故障排查

**问题：修改后页面没有更新**
- 解决：重启开发服务器 `npm run dev`

**问题：首页个人信息不显示**
- 检查 `profile.mdx` 中的 `type: profile` 是否正确
- 检查文件是否保存在 `content/basic/` 文件夹

**问题：社交链接不显示**
- 检查链接字段是否为空
- 检查链接格式是否正确
